
### 1 框架简介

如下是三个开源框架在GitHub主页上的介绍

**1.1 nndeploy**：nndeploy是一款模型端到端部署框架。以多端推理以及基于有向无环图模型部署为基础，致力为用户提供跨平台、简单易用、高性能的模型部署体验。

**1.2 fastdeploy**：FastDeploy是一款全场景、易用灵活、极致高效的AI推理部署工具， 支持云边端部署。提供超过 160+ Text，Vision， Speech和跨模态模型开箱即用的部署体验，并实现端到端的推理性能优化。包括 物体检测、字符识别（OCR）、人脸、人像扣图、多目标跟踪系统、NLP、Stable Diffusion文图生成、TTS 等几十种任务场景，满足开发者多场景、多硬件、多平台的产业部署需求。

**1.2 mmdeploy**：MMDeploy 是 OpenMMLab 模型部署工具箱，为各算法库提供统一的部署体验。基于 MMDeploy，开发者可以轻松从训练 repo 生成指定硬件所需 SDK，省去大量适配时间。

### 2 框架对比

| 名称       | 易用性 | 性能 | 并行   | 开放性 | 模型支持 | 扩展性 |
| ---------- | ------ | ---- | ------ | ------ | -------- | ------ |
| nndeploy   | 高     | 高   | 支持   | 高     | 低       | 低     |
| fastdeploy | 中     | 中   | 不支持 | 中     | 高       | 高     |
| mmdeploy   | 中     | 中   | 不支持 | 中     | 高       | 高     |

**2.1 易用性**：基于有向无环图部署模型，模型部署的全流程（前处理->推理->后处理）简单明了；构建推理模板Infer，可屏蔽模型的复杂性；高效解决多模型的复杂场景，通过支持图中嵌入图灵活且强大的功能，将大问题拆分为小问题，通过组合的方式快速解决多模型的复杂场景问题；可快速构建demo

**2.2 性能**：对推理框架的高性能抽象；高效的线程池

**2.3 并行**：基于有向无环图部署模型，支持多种并行模式，进一步提升模型部署的性能

**2.4 开放性**：目前fastdeploy重点是paddle的模型仓库以及主推自家推理框架，mmdeploy主要是openmmlab的模型仓库以及主推自家推理框架，而nndeploy积极集成各家的推理框架，不设限的部署热门的开源模型。nndeploy想跳出这个"特定优先支持"，做一个更加通用普适的部署框架

**2.5 模型支持**：目前nndeploy支持模型较少；fastdeploy支持paddle的模型仓库中绝大部分模型；mmdeploy支持openmmlab的模型仓库中绝大部分模型

**2.6 扩展性**：nndeploy有更加良好的架构，可扩展为推理框架，也可扩展支持分布式的推理部署


## 竞品

### 3.1 模型部署框架

**3.1.1 fastdeploy**：FastDeploy 是百度 PaddlePaddle 推出的全场景、易用灵活、极致高效的 AI 推理部署工具，支持云边端部署。提供超过 160+ Text、Vision、Speech 和跨模态模型开箱即用的部署体验，并实现端到端的推理性能优化。包括物体检测、字符识别（OCR）、人脸、人像扣图、多目标跟踪系统、NLP、Stable Diffusion 文图生成、TTS 等几十种任务场景。主要特点：重点支持 Paddle 模型仓库和自家推理框架，模型支持数量多，但开放性相对较低。

**3.1.2 mmdeploy**：MMDeploy 是 OpenMMLab 模型部署工具箱，为各算法库提供统一的部署体验。基于 MMDeploy，开发者可以轻松从训练 repo 生成指定硬件所需 SDK，省去大量适配时间。主要特点：主要支持 OpenMMLab 的模型仓库和自家推理框架，模型支持数量多，但同样存在开放性限制。

**3.1.3 triton inference server**：NVIDIA Triton Inference Server 是一个开源的推理服务框架，旨在简化在生产环境中部署 AI 模型的过程。主要特点：侧重于对模型推理阶段的统一管理，屏蔽了内部推理框架接口，注重对用户的推理框架黑盒化，优化主要集中在推理阶段的资源/任务调度。与 nndeploy 的区别在于：triton 关注推理阶段，nndeploy 关注端到端部署（包括前后处理）；triton 面向普通用户，nndeploy 面向开发人员并保留对推理框架的控制能力。

**3.1.4 mediapipe**：Google MediaPipe 是一个用于构建感知管道的框架，用于处理视频、音频等时间序列数据。主要特点：提供预构建的解决方案（如人脸检测、手部跟踪、姿态估计等），支持跨平台部署（Android、iOS、Web、桌面），采用图结构管理数据处理流程。与 nndeploy 的区别：MediaPipe 主要面向感知任务，提供预构建的解决方案；nndeploy 更通用，支持任意模型的端到端部署。

**3.1.5 comfyui**：ComfyUI 是一个基于节点的工作流界面，用于 Stable Diffusion 等生成模型的部署和使用。主要特点：可视化节点式工作流设计，支持复杂的生成流程编排（文本编码、噪声调度、UNet 推理、VAE 解码等），灵活的参数控制和实时预览。与 nndeploy 的区别：ComfyUI 主要面向 Stable Diffusion 等生成模型，是应用层面的工具；nndeploy 是通用的模型部署框架，支持多种模型类型和推理后端。

**3.1.6 nexa-sdk**：Nexa SDK 是一个 AI 模型部署框架，提供模型推理和管理能力。主要特点：支持多种推理后端，提供统一的 API 接口，支持模型版本管理和动态加载。

**3.1.7 TEN-framework**：TEN（Tensor Engine Network）是一个高性能的深度学习推理框架。主要特点：专注于推理性能优化，支持多种硬件平台，提供高效的算子实现和内存管理。

### 3.2 Agent 和工作流编排框架

**3.2.1 coze**：字节跳动 Coze 是一个 AI 应用开发平台，支持可视化工作流编排和 Agent 构建。主要特点：提供可视化界面构建 AI 应用，支持多模型组合和复杂工作流编排，集成多种 AI 能力（对话、知识库、插件等）。与 nndeploy 的区别：Coze 是应用层平台，主要面向 LLM 应用开发；nndeploy 是底层部署框架，专注于模型推理部署。

**3.2.2 dify**：Dify 是一个开源的 LLM 应用开发平台，支持工作流编排和 Agent 构建。主要特点：可视化工作流编辑器，支持拖拽式构建 AI 应用，提供 RAG、Agent、工作流等多种应用模式，支持多种 LLM 模型接入。与 nndeploy 的区别：Dify 主要面向 LLM 应用开发，提供应用层能力；nndeploy 专注于模型部署，支持多种模型类型和推理后端。

**3.2.3 langgraph**：LangGraph 是 LangChain 的低级编排框架，用于构建长时间运行的有状态智能体。主要特点：基于状态机的工作流设计，支持持久执行和人机协作，提供全面的记忆管理，支持复杂条件分支和可视化调试。与 nndeploy 的区别：LangGraph 专注于 Agent 编排和 LLM 应用，采用状态机模式；nndeploy 专注于模型部署，采用有向无环图模式。

**3.2.4 autogen**：AutoGen 是微软官方的多 Agent 协作框架，支持多智能体协作和可视化编排。主要特点：天然支持多智能体协作，提供 AutoGen Studio 可视化编排工具，支持代码执行环境，企业级支持。与 nndeploy 的区别：AutoGen 专注于多 Agent 协作和 LLM 应用；nndeploy 专注于模型部署和多模型组合。

**3.2.5 crewai**：CrewAI 是一个角色导向的 Agent 框架，API 设计简洁易用。主要特点：清晰的角色定义，快速开发能力（1小时搭建原型），适合快速验证想法和小规模应用。与 nndeploy 的区别：CrewAI 专注于 Agent 协作和 LLM 应用；nndeploy 专注于模型部署。

**3.2.6 langflow**：Langflow 是一个用于构建多代理和 RAG 应用的可视化框架。主要特点：拖放式界面，无需大量编码，支持多代理编排，完全开源且基于 Python。与 nndeploy 的区别：Langflow 专注于 LLM 应用和 RAG 场景；nndeploy 是通用的模型部署框架。

### 3.3 竞品对比总结

| 类别 | 框架 | 定位 | 与 nndeploy 的主要区别 |
|------|------|------|----------------------|
| 模型部署 | fastdeploy | Paddle 生态部署工具 | 重点支持 Paddle 模型，开放性较低 |
| 模型部署 | mmdeploy | OpenMMLab 部署工具 | 重点支持 OpenMMLab 模型，开放性较低 |
| 模型部署 | triton | 推理服务框架 | 关注推理阶段，面向普通用户 |
| 模型部署 | mediapipe | 感知管道框架 | 面向感知任务，提供预构建方案 |
| 模型部署 | comfyui | Stable Diffusion 工作流工具 | 面向生成模型，应用层工具 |
| Agent 编排 | coze | AI 应用开发平台 | 应用层平台，面向 LLM 应用 |
| Agent 编排 | dify | LLM 应用开发平台 | 应用层平台，面向 LLM 应用 |
| Agent 编排 | langgraph | Agent 编排框架 | 专注于 Agent 编排，采用状态机模式 |
| Agent 编排 | autogen | 多 Agent 协作框架 | 专注于多 Agent 协作 |
| Agent 编排 | crewai | Agent 快速开发框架 | 专注于 Agent 协作 |
| Agent 编排 | langflow | 可视化流构建框架 | 专注于 LLM 应用和 RAG |

**nndeploy 的核心优势**：
- **定位清晰**：专注于模型端到端部署，而非应用层开发
- **开放性强**：不绑定特定模型仓库，支持各推理框架
- **性能优化**：端到端性能优化，包括前后处理和推理
- **通用性强**：支持多种模型类型，不限于特定场景
- **开发友好**：保留对推理框架的控制能力，适合高级用户
